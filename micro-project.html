// Process

How the System Works

From raw air samples to real-time readings — the complete signal pipeline of the AirSense monitor.

01
Sensing

Air & Environment Sampling

The PMS5003 laser particle sensor draws in ambient air and counts particles by size, outputting PM1.0, PM2.5, and PM10 concentrations. Simultaneously the BME280 reads temperature, relative humidity, and barometric pressure over I²C. Both sensors run continuously at their native polling rates, providing a complete snapshot of air quality every second.

PMS5003BME280I²C Bus
02
Programming

Flashing & Debug via FTDI

The FTDI USB-TTL module bridges the computer's USB port to the MCU's UART TX/RX pins. This allows firmware to be uploaded using the Arduino IDE or PlatformIO, and serial output to be monitored in real time for debugging. The FTDI also enables configuring the Wi-Fi module by sending AT commands from a terminal window.

FTDI USB-TTLUARTAT Commands
03
Processing

Microcontroller Data Handling

The MCU reads PMS5003 data via software serial UART and BME280 data via I²C. It performs unit conversions, AQI index calculations, and threshold comparisons. Results are distributed simultaneously to three output paths: the OLED display, alert system (buzzer + LED), and wireless module — all orchestrated in a tight main loop running every second.

MCU CoreUART + I²CMini BreadboardJumper Wires
04
Display

OLED Live Readout

The 0.96" SSD1306 OLED (128×64px, I²C) renders a fixed-width text dashboard refreshed every second. PM2.5, PM10, temperature, humidity, and pressure readings are shown in structured rows. Inverted pixel blocks visually flag dangerous readings — all without a backlight, keeping power consumption minimal for battery operation.

0.96" OLED SSD1306I²C 0x3C
05
Alerts

Buzzer & LED Threshold Alerts

When PM2.5 exceeds 35 µg/m³ (WHO unsafe threshold), the Active Buzzer fires a short beep pattern and the LED illuminates. The LED's current-limiting resistor (220–470Ω) protects the GPIO pin. These dual alerts enable background monitoring — no need to watch the OLED for hazardous air quality events.

Active BuzzerLED + ResistorGPIO Digital Out
06
Wireless

Wi-Fi Data Transmission

The Wi-Fi module receives formatted sensor data from the MCU over serial and publishes it to an MQTT broker or HTTP REST endpoint every 10 seconds. A companion web dashboard plots live charts and logs historical readings. Bluetooth mode allows the device to stream data directly to a paired smartphone without any router in range.

Wi-Fi ModuleMQTT / HTTPBluetooth
07
Power

Battery Power & Charging

An 18650 Li-ion cell supplies 3.7V. The TP4056 charging module manages safe CC/CV charging via Micro-USB, with built-in overcharge, over-discharge, and short-circuit protection. A boost converter steps voltage up where 5V components require it, making the device fully portable and field-ready.

// Principles

How Each Technology Works

The underlying physics and engineering behind each core component in this circuit.

💨
Laser Particle Counting
PMS5003 — Mie scattering physics

The PMS5003 uses laser light scattering (Mie scattering): a fan pulls ambient air through a dark chamber where a laser beam illuminates passing particles. Each particle deflects light onto a photodetector. The resulting electrical pulse duration and peak voltage correspond directly to particle size, letting firmware count and classify PM1.0, PM2.5, and PM10 concentrations in µg/m³.

🌡️
MEMS Environmental Sensing
BME280 — Three transducers, one chip

The BME280 uses three MEMS transduction mechanisms on a single silicon die. Pressure: a microscopic silicon diaphragm deflects under air pressure, read as a capacitance change. Temperature: a resistive thermistor bridge changes resistance with heat. Humidity: a capacitive polymer layer absorbs water vapor, altering its dielectric constant. All three are digitized on-chip with 16–20 bit ADCs and factory-calibrated.

🖥️
Organic LED Display
SSD1306 — Self-emitting pixel matrix

Unlike LCD panels, OLED pixels emit their own light — each is a tiny organic electroluminescent sandwich. When a forward voltage is applied, holes and electrons recombine in the organic emissive layer, releasing photons. The SSD1306 manages charge injection into the 128×64 matrix via row/column drivers over I²C. Pixels set to black draw zero current — no backlight, no wasted power — ideal for battery-powered embedded displays.

🔋
Lithium-Ion Electrochemistry
18650 + TP4056 — CC/CV charge management

Li-ion cells store energy through intercalation: during charging, Li⁺ ions migrate from the LiCoO₂ cathode through electrolyte into the graphite anode lattice. The TP4056 implements Constant Current / Constant Voltage (CC/CV) charging — first pushing full current while voltage rises, then switching to constant 4.2V while current tapers to ~10% of rated value, signalling full charge and maximising capacity without cell damage.

📡
802.11 Wi-Fi Radio
ESP Module — OFDM packet transmission

The ESP module uses OFDM (Orthogonal Frequency-Division Multiplexing) on the 2.4 GHz ISM band. Sensor data is broken into TCP/IP packets, modulated onto orthogonal subcarriers, and sent to a router. MQTT adds a lightweight publish/subscribe layer on top — the device publishes to a topic, and any subscribed client (dashboard, phone) receives updates instantly. Bluetooth uses FHSS for short-range secure pairing.

🔴
Alert Output Circuitry
Buzzer + LED — Ohm's Law in action

The active buzzer houses its own RC oscillator — voltage applied produces a fixed ~2.4 kHz tone with zero MCU overhead, no PWM required. The LED requires a series resistor calculated by Ohm's Law: R = (Vcc − Vf) / If. For 5V supply and red LED (Vf ≈ 2.0V, If = 15mA): R = (5−2)/0.015 = 200Ω → use standard 220Ω. This keeps GPIO current well below the MCU's 40mA maximum pin rating.
